Working with Docker

Full details of installing, configuring and using Docker can be found on the Docker Documentation web site

The following sections provide basic examples for building and running docker images for the published stubs. For more detailed information, see the IBM Rational Integration Tester Knowledge Center

Building a docker image that contains the stubs

The following basic docker command could be used to build an image from the Dockerfile and context:

docker build -t <myimage> <path_to_publish_output_directory>

where:
<myimage> is the name you would like to give to the new image
<path_to_publish_output_directory> is the directory where the Dockerfile and related files were published

For example:

docker build -t mystub /output

Running a container to run the stubs

The following basic docker command could be used to run the built image:

docker run -e RATIONAL_LICENSE_FILE=<license_server> <myimage>

where:
<license_server> is the host and port of the rational common license server in the form port@host
<myimage> is the name of the image you previously created

For example:

docker run -e RATIONAL_LICENSE_FILE=27000@licserver.example.com mystub

Note
If you require the stub to work with the Rational Integration Tester HTTP/TCP proxy you will need to specify further options when running the container. For information see IBM Rational Integration Tester Knowledge Center
